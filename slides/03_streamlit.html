O que é

Streamlit é
Python que vira web.

Você escreve Python puro. O Streamlit converte automaticamente em uma página web com botões, campos, gráficos e tabelas.

Sem HTML. Sem CSS. Sem JavaScript. Sem servidor configurado.

ANTES — PARA TER UMA WEB APP
  • ✗ Aprender HTML + CSS + JavaScript
  • ✗ Configurar servidor Flask ou FastAPI
  • ✗ Deploy manual em servidor pago
  • ✗ Semanas de trabalho
AGORA — COM STREAMLIT
  • ✓ Só Python — linguagem que você já usa
  • ✓ Um arquivo .py → app completo
  • ✓ Deploy gratuito via GitHub
  • ✓ Minutos, não semanas
Como funciona

O modelo de
execução.

A regra mais importante do Streamlit — e a que mais confunde iniciantes:

Toda vez que o usuário interage, o script inteiro roda de novo do início.

👆
Usuário interage
Clica num botão, move um slider, digita um valor
→
🔄
Script roda do início
Python executa do topo ao fundo novamente
→
🖥️
Tela atualiza
Resultado novo aparece instantaneamente
→
✓
Sem recarregar
O browser não pisca. É reativo e rápido.

Consequência prática: variáveis não persistem entre interações. Para guardar estado, usa-se st.session_state.

Componentes — Parte 1

Entrada de
dados.

Número
st.number_input()
Campo numérico com min, max e step. Ideal para área, volume, custo.
Texto
st.text_input()
Campo de texto livre. Nome do engenheiro, descrição, obra.
Seleção
st.selectbox()
Menu dropdown. Tipo de solo, equipamento, padrão construtivo.
Múltipla
st.multiselect()
Seleção de múltiplos itens. Serviços de uma obra, por exemplo.
Slider
st.slider()
Barra deslizante. Percentual de BDI, fator de correção.
Botão
st.button()
Dispara ação. Calcular, gerar relatório, limpar campos.
Chave
st.toggle()
Liga/desliga. Incluir BDI, mostrar detalhes, modo avançado.
Upload
st.file_uploader()
Carrega arquivo. CSV de preços, PDF de sondagem, planilha.
Componentes — Parte 2

Saída e
visualização.

Texto
st.write()
Exibe qualquer coisa: texto, número, DataFrame, gráfico.
Métrica
st.metric()
Número em destaque com delta. R$ 124.500 ↑ 12%. Perfeito para KPIs de obra.
Tabela
st.dataframe()
Exibe DataFrame Pandas interativo com filtro e ordenação.
Gráfico
st.plotly_chart()
Gráficos Plotly interativos. Curva S, histograma, dispersão.
Sucesso
st.success()
Caixa verde de confirmação. "Cálculo concluído com sucesso."
Alerta
st.warning()
Caixa amarela de atenção. "Volume acima do esperado para o solo."
Download
st.download_button()
Exporta resultado em CSV, Excel ou PDF direto do browser.
Layout
st.columns()
Divide a tela em colunas. Inputs à esquerda, resultado à direita.
Componentes — Parte 3

Estrutura e
navegação.

Abas
st.tabs()
Divide em abas. Entrada / Resultado / Sobre. Organiza app complexo.
Lateral
st.sidebar
Painel lateral fixo. Configurações, filtros globais, parâmetros.
Expansor
st.expander()
Seção recolhível. Mostrar memória de cálculo, detalhes técnicos.
Container
st.container()
Agrupa elementos. Controla onde cada bloco aparece na tela.
Cabeçalho
st.title() / st.header()
Títulos hierárquicos. Nome do app, seção, subseção.
Divisor
st.divider()
Linha horizontal. Separa inputs de outputs visualmente.
Spinner
st.spinner()
Loading animado. Enquanto o cálculo pesado roda.
Multipage
pages/
Múltiplas páginas. Cada .py na pasta pages vira uma aba no menu lateral.
Estrutura mínima

O esqueleto de
todo app.

import streamlit as st

# 1. TÍTULO DO APP
st.title("Calculadora de Escavação")

# 2. ENTRADAS DO USUÁRIO
volume   = st.number_input("Volume (m³)", min_value=0.0)
tipo     = st.selectbox("Tipo de solo", ["argiloso", "rochoso"])
equip    = st.selectbox("Equipamento", ["escavadeira_20t", "retro"])

# 3. LÓGICA PYTHON (sua função de engenharia)
if st.button("Calcular"):
    horas = calcular_escavacao(volume, tipo, equip)
    st.metric("Horas estimadas", f"{horas:.1f} h")
    st.success("Cálculo concluído.")

Três blocos: título → entradas → lógica + saída. Todo app de engenharia segue esse padrão.

# aula02_hello_engenharia.py — já no repositório

def calcular_escavacao(volume_m3, tipo_solo, equipamento):
    produtividade = {
        "argiloso": {"escavadeira_20t": 45, "retro": 18},
        "rochoso":  {"escavadeira_20t": 12, "retro":  5},
    }
    horas = volume_m3 / produtividade[tipo_solo][equipamento]
    return round(horas, 2)
