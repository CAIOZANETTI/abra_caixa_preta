Antes de tudo

Primeiro o
repositório.

Todo projeto começa no GitHub — não no código.
O repositório é sua nuvem, seu portfólio e sua base.
Sem ele, o código que a IA gera não vai a lugar nenhum.

github.com/CAIOZANETTI/abra_caixa_preta
Esse é o repositório desta aula — público, auditável, para sempre.

Prática — GitHub

Criar o
repositório.

01
Acesse github.com/new
Logue na sua conta e clique em New repository. É daqui que tudo começa.
02
Nome: abra_caixa_preta
Use underscore, sem espaço. Deixe Public. Esse repositório é seu portfólio — precisa ser visível.
03
Ative o README + .gitignore Python
README é o contrato do projeto. .gitignore Python evita subir lixo. License: sem licença por enquanto.
04
Clique em Create repository
Pronto. Sua nuvem está no ar. URL pública gerada. Esse link vai para o aluno da Hotmart.

01
Ferramenta 01

Codex

OpenAI Codex
platform.openai.com/docs/guides/code
Via API Base do Copilot Pay per use
  • Motor por trás do GitHub Copilot
  • Acesso via API com tokens
  • Ótimo para autocomplete inline
  • Menos raciocínio, mais velocidade
  • Integra com qualquer editor
  • Ideal para snippets repetitivos

Melhor para: autocompletar código repetitivo, boilerplate, scripts rápidos sem análise profunda.

02
Ferramenta 02

Cursor

Cursor IDE
cursor.sh
IDE Completa Plano Free Baseado em VSCode
  • VSCode com IA embutida
  • Vê o projeto inteiro como contexto
  • Edita múltiplos arquivos de uma vez
  • Modo Composer para refatoração
  • Usa Claude ou GPT por baixo
  • Melhor custo-benefício do mercado

Melhor para: projetos com vários arquivos, refatoração contínua, desenvolvimento ativo.

03
Ferramenta 03

Antigravity

Antigravity
antigravity.dev
Agente autônomo Plano Free IDE própria
  • Agente que navega no projeto sozinho
  • Executa sequência de edições em cascata
  • Roda comandos no terminal automaticamente
  • Mais autônomo que o Cursor
  • Plano gratuito bastante generoso
  • Boa opção para quem está começando

Melhor para: tarefas longas e repetitivas, scaffolding de projeto, estruturação inicial.

04
Ferramenta 04 — Nossa escolha

Claude

Claude — Anthropic
claude.ai
Chat + API + CLI Mais analítico 200k tokens
  • Raciocina antes de escrever código
  • Explica cada decisão técnica
  • Contexto de 200k tokens (projetos grandes)
  • Claude Code: agente no terminal
  • Melhor para lógica de engenharia civil
  • Auditável — você entende o que fez

Passo 02 — Claude
Abra o claude.ai e cole o prompt

No Claude Chat, você vai passar o contexto do projeto e pedir o primeiro código Python que rode direto no prompt.

claude.ai → nova conversa
Prática — O Prompt

Peça o primeiro
código para o Claude.

Meu repositório é github.com/CAIOZANETTI/abra_caixa_preta

Crie um arquivo Python chamado aula02_hello_engenharia.py
que rode no prompt sem nenhuma dependência externa.

O código deve:
— Pedir o nome do engenheiro
— Pedir a área da obra em m²
— Calcular um custo estimado simples (R$ 2.500/m²)
— Mostrar o resultado formatado com separador de milhar

Explique cada linha com comentário em português.

Isso é o que faremos ao vivo agora → Claude gera, você copia, roda no terminal, vê funcionar.
